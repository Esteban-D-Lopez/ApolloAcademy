Key architectural decision: The frontend is almost entirely self-contained. All curriculum, terms, citations, and sources are static JS data files — no API needed to browse lessons, search, or take quizzes. Only the Chat page hits the backend. This means the app is fast, works offline for 12/13 pages, and the backend is only needed for AI-powered Q&A.
v2 — Multi-provider LLM: Users can select Google Gemini, OpenAI, or Anthropic as the chat model and enter their own API key directly in the Chat UI. The key is stored in localStorage only and sent to localhost:8000 — never bundled or committed. LangGraph's graph structure is identical for all providers; only the LLM node swaps out.

4 RAG Chat — Full Request/Response Flow

1
User types a question in Chat.jsx
Input captured in React state. On submit, the last 20 messages are assembled into the payload (history capped to keep API costs bounded).
POST /api/chat { messages: [...last 20] }
2
FastAPI validates the request (server.py)
Pydantic enforces: role must be "user" or "assistant", content ≤ 4,000 chars, 1–50 messages. Invalid requests are rejected with HTTP 422 before any LLM call.
3
LangGraph agent invoked (agent.py)
Messages converted to LangChain format (HumanMessage / AIMessage). System prompt prepended. Graph enters the agent node.
graph.invoke({ messages: lc_messages })
4
Selected LLM decides to call search_documents
The active provider's LLM (Gemini / GPT-4o / Claude, temp=0.1) inspects the question and emits a tool call. The system prompt mandates it ALWAYS searches before answering — this rule applies to all providers.
search_documents(query="...", k=6) # embeddings always use Gemini (index compatibility)
5
ChromaDB vector search (tools.py)
Query embedded with Gemini Embeddings. Cosine similarity search returns top-6 chunks with metadata (document name, page, source_type, relevance score).
[Result 1] (Relevance: 0.87) Source: 2026 10-K.pdf (p. 42) [SEC Filing] Content: "Apollo's fee-related earnings..."
6
Gemini generates a cited answer
Tool results injected back. Gemini synthesizes a structured response with inline citations in the format [Source: doc.pdf, p. N].
7
FastAPI extracts citations + responds
Regex parses [Source: ...] markers into a structured citations list. Response JSON returned to the browser.
{ response: "...", citations: [ { document, page, excerpt }, ... ]}

7 Security

🛡️
XSS Prevention — Safe React Renderer
Chat messages used to use dangerouslySetInnerHTML. Now Chat.jsx parses markdown (bold, headers, bullets, citation chips) into real React elements via parseInline() + renderContent(). React escapes all text content automatically — no raw HTML injection possible.
🔑
API Key Handling — Two Layers
Backend default: GOOGLE_API_KEY in backend/.env (gitignored) is used when no user key is provided.
User-provided key (v2): Users enter their own Gemini / OpenAI / Anthropic key in the Chat settings UI. It is stored in localStorage only — never bundled, never committed, never sent anywhere except localhost:8000. A prominent warning in the UI makes clear this is for local use only. The key is transmitted in the HTTPS request body to the local server and used for that request only.
✅
Input Validation — Backend
Pydantic enforces role: Literal["user", "assistant"], content: max_length=4000, and messages: 1–50 items. Invalid payloads get HTTP 422 before reaching the LLM — prevents prompt injection via oversized inputs and rejects unknown roles.
📡
CORS Policy
Backend only allows origins localhost:5173, localhost:5174, and localhost:3000. In production this list should be updated to the deployed frontend domain only.
💬
Bounded Chat History
Only the last 20 messages are sent to the backend per request. The full conversation is displayed in the UI but the API payload is capped — prevents unbounded token costs and oversized request bodies.
